📚 Dokumentation – LLMWikiNG

Version {{ app_version }} · Vollständiges Handbuch für Einrichtung, Multi-Wiki, Authentifizierung und API.

🚀 Erste Schritte

  1. Abhängigkeiten installieren: pip install -r requirements.txt
  2. Server starten: ./start.sh (oder python3 run.py --port 8080)
  3. Beim ersten Start (Setup) wirst du automatisch auf /register umgeleitet, um den ersten Benutzer (Administrator) anzulegen.
  4. Im Browser öffnen: http://localhost:8080/LLMWikiNG/

Das ursprüngliche wiki/ wird beim ersten Start automatisch nach wikis/main/ verschoben (Daten bleiben erhalten).

🗂️ Mehrere Wikis (Unter-Wikis)

Jedes Wiki liegt in wikis/<name>/ und hat einen eigenen Namen. Über die WebUI lässt sich ein neues Wiki anlegen:

  • /LLMWikiNG/wikis/new – Wiki über das Formular erstellen.
  • Wiki-Startseite: /LLMWikiNG/wiki/<Name>/
  • Wiki-Seite: /LLMWikiNG/wiki/<Name>/<Seite>

Der Basis-Pfad ist über die Umgebungsvariable LLMWIKI_BASE_PATH konfigurierbar (Standard: /LLMWikiNG), z. B. für example.com/LLMWikiNG/wiki/<Name>/.

🔐 Authentifizierung & Benutzerverwaltung

  • Login mit Benutzername + Passwort (Argon2-Hashes, signierte Sessions).
  • Registrierung über /register (wird nach dem ersten Benutzer automatisch deaktiviert).
  • /LLMWikiNG/users – Benutzer anlegen/löschen (nur Admin).
  • /LLMWikiNG/api-keys – API-Keys verwalten (nur Admin).
  • Ein API-Key kann optional verlangen, dass zusätzlich ein Passwort (X-API-Password) mitgesendet wird.

⚙️ Einstellungen (Settings-Tabs)

Die zentrale Konfigurationsseite /LLMWikiNG/settings ist in Tabs unterteilt:

  • Sprache – Sprachumschaltung (DE/EN, Cookie).
  • Erscheinungsbild – Theme Dark/Light, persistent in config.json gespeichert.
  • SMTP – E-Mail-Konfiguration für Wochenberichte/Briefings (in config.json).
  • Benutzerverwaltung – Benutzer anlegen/löschen, Rollen (Admin/Editor/Viewer).
  • API-Schlüssel – API-Keys erzeugen/widerrufen, optional mit Passwort-Pflicht.
  • Registrierung – Einstellen ob die Benutzerregistrierung unter /register aktiv ist.
  • Gesundheitscheck – Linter (Orphans, fehlende Links, Staleness, Raw-Refs).
  • Update – Selbstupdate via GitHub.

🌐 API (JSON, Key-geschützt)

Alle Funktionen sind unter /LLMWikiNG/api/v1 per X-API-Key-Header erreichbar.

📋 cURL-Befehlsbeispiele (Fernbedienung)

Exportiere deine Variablen, um die Befehle direkt zu kopieren:

export API_KEY="llmw_dein_key"
export SERVER_URL="http://localhost:8082/LLMWikiNG"
export WIKI="main"
1. Wikis auflisten: 
curl -H "X-API-Key: $API_KEY" "$SERVER_URL/api/v1/wikis"
2. Wiki-Seiten auslesen: 
curl -H "X-API-Key: $API_KEY" "$SERVER_URL/api/v1/wikis/$WIKI/pages"
3. Spezifische Seite abrufen: 
curl -H "X-API-Key: $API_KEY" "$SERVER_URL/api/v1/wikis/$WIKI/pages/llm-wiki"
4. Neue Seite erstellen (YAML-Frontmatter-Pflicht): 
curl -X POST -H "X-API-Key: $API_KEY" -H "Content-Type: application/json" \
  -d '{"slug":"api-test","content":"---\ntype: Concept\ntitle: API Test\ntimestamp: 2026-07-14T22:00:00Z\n---\n# API Test\nErfolgreich erstellt!"}' \
  "$SERVER_URL/api/v1/wikis/$WIKI/pages"
5. Direkter Multi-Wiki-Ingest (Datei, URL oder Text): 
# URL-Ingest
curl -X POST -H "X-API-Key: $API_KEY" -d "url=https://de.wikipedia.org/wiki/Google" -d "title=Google" "$SERVER_URL/wiki/$WIKI/api/ingest"

# Datei-Upload
curl -X POST -H "X-API-Key: $API_KEY" -F "file=@/pfad/zur/datei.txt" "$SERVER_URL/wiki/$WIKI/api/ingest"

# Text-Paste
curl -X POST -H "X-API-Key: $API_KEY" -d "text=# Mein Text\nInhalt..." -d "title=Text-Paste" "$SERVER_URL/wiki/$WIKI/api/ingest"
6. Volltextsuche: 
curl -H "X-API-Key: $API_KEY" "$SERVER_URL/api/v1/search?q=Suchbegriff&wiki=$WIKI"
7. Wissensgraph-Netzwerk abrufen: 
curl -H "X-API-Key: $API_KEY" "$SERVER_URL/api/v1/wikis/$WIKI/graph"
8. Benutzer anlegen (nur Admin): 
curl -X POST -H "X-API-Key: $API_KEY" -H "Content-Type: application/json" \
  -d '{"username":"neuer_user","password":"sicheres_passwort","role":"editor"}' \
  "$SERVER_URL/api/v1/users"
9. API-Keys auflisten & generieren (nur Admin): 
# Listen
curl -H "X-API-Key: $API_KEY" "$SERVER_URL/api/v1/api-keys"

# Erzeugen (optional mit require_password: true)
curl -X POST -H "X-API-Key: $API_KEY" -H "Content-Type: application/json" \
  -d '{"name":"Neuer Key","require_password":false,"scopes":["read","write"]}' \
  "$SERVER_URL/api/v1/api-keys"

🔑 Kryptografisches Geheimnis (LLMWIKI_SECRET)

Das System verwendet ein zentrales Geheimnis, um Browser-Sitzungscookies zu signieren und API-Schlüssel zu verschlüsseln. Es wird aus der Umgebungsvariablen LLMWIKI_SECRET ausgelesen oder beim ersten Start automatisch generiert und dauerhaft in der Datei config.json (unter dem Schlüssel "secret_key") gespeichert.

⚠️ Warnung zur Rotation des Geheimnisses:
Das Ändern des System-Geheimnisses führt zum sofortigen Ablauf aller aktiven Web-Sitzungen (erfordert eine erneute Anmeldung) und macht zuvor verschlüsselte API-Schlüssel unlesbar. Nach dem Ändern des Secrets müssen alte API-Keys gelöscht und in der Administration neu erstellt werden.
  • Verwaltung im WebUI: Als Administrator anmelden ➜ Einstellungen ➜ Backup & Restore ➜ Passwort eingeben zum Anzeigen oder Neu generieren.
  • Wiederherstellung bei Updates: Wenn Sie eine bestehende Instanz aktualisieren, kopieren Sie das alte Geheimnis aus der vorherigen Konfiguration und tragen es als Wert für "secret_key" in der config.json ein.

🌗 Erscheinungsbild (Theme)

Das Aussehen (Dark/Light) wird ausschließlich in den Einstellungen gesteuert und persistent in config.json gespeichert ("theme": "dark" bzw. "light"). Dark Mode ist Standard und wird server-seitig geladen – es gibt keine Umschalter-Buttons in der Sidebar.

  • /LLMWikiNG/settings?tab=theme – Theme über das Tab „Erscheinungsbild“ ändern.
  • POST /LLMWikiNG/theme/set – direkt via Form-Feld value=dark|light (Login nötig).
  • Standard wiederherstellen: "theme" in config.json auf dark setzen.

🐋 Docker & UGreen NAS

Das Docker-Setup ist speziell für UGreen NAS-Systeme (UGOS) optimiert, um eine reibungslose Kompatibilität zu gewährleisten. Die Ordnerstruktur wird dabei automatisch unter volume1/docker/llmwiking/ abgelegt.

UGreen NAS (UGOS) Einrichtung:

Beim Importieren des Images in den UGreen Container Manager werden folgende Volumes vordefiniert:

  • /app/data ➜ /volume1/docker/llmwiking/data (Benutzer und API-Keys)
  • /app/wikis ➜ /volume1/docker/llmwiking/wikis (Wiki-Inhalte)
  • /app/raw ➜ /volume1/docker/llmwiking/raw (Rohquellen)
  • /app/output_docs ➜ /volume1/docker/llmwiking/output_docs (Dokument-Exporte)

Lokale Ausführung & Entwicklung:

Um das Setup auf einem anderen Host (z. B. lokal) auszuführen, ohne dass Mount-Fehler (z. B. durch config.json) auftreten, kann die Umgebungsvariable DOCKER_VOLUME_BASE genutzt werden:

  1. Erstelle eine lokale .env-Datei im Projektverzeichnis: 
    DOCKER_VOLUME_BASE=.
  2. Starte den Container: 
    docker compose up --build
