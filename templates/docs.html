📚 Documentation – LLMWikiNG

Version {{ app_version }} · Complete manual for setup, multi-wiki, authentication, and API.

🚀 Getting Started

  1. Install dependencies: pip install -r requirements.txt
  2. Start the server: ./start.sh (or python3 run.py --port 8080)
  3. On the first start (setup), you will be redirected automatically to /register to create the initial admin user.
  4. Open in browser: http://localhost:8080/LLMWikiNG/

The original wiki/ folder is migrated to wikis/main/ automatically upon the first startup (no data loss).

🗂️ Multiple Wikis (Sub-wikis)

Each wiki resides under wikis/<name>/ and has its own display name. Use the WebUI to create new wikis:

  • /LLMWikiNG/wikis/new – Form to create a new wiki.
  • Wiki Homepage: /LLMWikiNG/wiki/<Name>/
  • Wiki Page: /LLMWikiNG/wiki/<Name>/<Page>

The app's base URL path can be changed via the LLMWIKI_BASE_PATH environment variable (default: /LLMWikiNG), e.g. example.com/LLMWikiNG/wiki/<Name>/.

🔐 Authentication & User Management

  • Secure login with username and password (Argon2 hashes, signed session cookies).
  • Registration via /register (auto-disables after the first user is created).
  • /LLMWikiNG/users – Add/delete users (Admin only).
  • /LLMWikiNG/api-keys – Manage API Keys (Admin only).
  • An API Key can optionally require a password (X-API-Password header or query param) alongside it.

⚙️ Settings Panels

The central settings page /LLMWikiNG/settings is divided into tabs:

  • Language – Choose UI language (DE/EN, stored in cookies).
  • Appearance – Choose Theme (Dark/Light), stored persistently in config.json.
  • SMTP – Email configuration for weekly reports & briefings (stored in config.json).
  • User Management – Manage users and roles (Admin/Editor/Viewer).
  • API Keys – Generate or revoke API Keys, toggle password-requirement.
  • Registration – Turn public registration on or off.
  • Health Check – Run the wiki linter (orphans, missing targets, staleness, raw-file refs).
  • Wiki Management – Create, edit (inline), and delete wikis with page/file count and size.
  • Update – Pull updates directly from GitHub.

🌐 API (JSON, Key-protected)

All actions are available under /LLMWikiNG/api/v1 via the X-API-Key header or session cookie (automatic fallback for the web browser).

📋 cURL examples (Terminal remote control)

Export variables to quickly test these queries:

export API_KEY="llmw_your_key"
export SERVER_URL="http://localhost:8082/LLMWikiNG"
export WIKI="main"
1. List all active wikis: 
curl -H "X-API-Key: $API_KEY" "$SERVER_URL/api/v1/wikis"
2. List all pages inside a wiki: 
curl -H "X-API-Key: $API_KEY" "$SERVER_URL/api/v1/wikis/$WIKI/pages"
3. Read specific wiki page: 
curl -H "X-API-Key: $API_KEY" "$SERVER_URL/api/v1/wikis/$WIKI/pages/llm-wiki"
4. Write a new page (requires valid YAML frontmatter): 
curl -X POST -H "X-API-Key: $API_KEY" -H "Content-Type: application/json" \
  -d '{"slug":"api-test","content":"---\ntype: Concept\ntitle: API Test\ntimestamp: 2026-07-14T22:00:00Z\n---\n# API Test\nCreated successfully!"}' \
  "$SERVER_URL/api/v1/wikis/$WIKI/pages"
5. Direct multi-wiki Ingest (File, URL or Raw Text): 
# URL Ingest
curl -X POST -H "X-API-Key: $API_KEY" -d "url=https://de.wikipedia.org/wiki/Google" -d "title=Google" "$SERVER_URL/wiki/$WIKI/api/ingest"

# File upload
curl -X POST -H "X-API-Key: $API_KEY" -F "file=@/path/to/file.txt" "$SERVER_URL/wiki/$WIKI/api/ingest"

# Raw text paste
curl -X POST -H "X-API-Key: $API_KEY" -d "text=# Raw Text\nContent..." -d "title=Text-Paste" "$SERVER_URL/wiki/$WIKI/api/ingest"
6. Volltextsuche: 
curl -H "X-API-Key: $API_KEY" "$SERVER_URL/api/v1/search?q=SearchQuery&wiki=$WIKI"
7. Get network graph coords: 
curl -H "X-API-Key: $API_KEY" "$SERVER_URL/api/v1/wikis/$WIKI/graph"
8. Create user account (Admin only): 
curl -X POST -H "X-API-Key: $API_KEY" -H "Content-Type: application/json" \
  -d '{"username":"new_user","password":"secure_password","role":"editor"}' \
  "$SERVER_URL/api/v1/users"
9. Manage API Keys (Admin only): 
# List active keys
curl -H "X-API-Key: $API_KEY" "$SERVER_URL/api/v1/api-keys"

# Generate new key (optional require_password toggle)
curl -X POST -H "X-API-Key: $API_KEY" -H "Content-Type: application/json" \
  -d '{"name":"New Key","require_password":false,"scopes":["read","write"]}' \
  "$SERVER_URL/api/v1/api-keys"
10. Check & run system update (Admin only): 
# Check if update is available:
curl -H "X-API-Key: $API_KEY" "$SERVER_URL/api/v1/system/update/check"

# Run the update:
curl -X POST -H "X-API-Key: $API_KEY" "$SERVER_URL/api/v1/system/update/run"

🔑 Cryptographic Secret (LLMWIKI_SECRET)

The system uses a central key to sign browser session cookies and encrypt API keys. It is read from the LLMWIKI_SECRET environment variable or dynamically generated and persisted inside config.json (under the key "secret_key") upon the first launch.

⚠️ Warning regarding Secret Rotation:
Rotating the secret key will immediately invalidate all active web sessions (forcing a re-login) and make previously encrypted API keys un-decryptable. After changing the secret, old API keys must be deleted and recreated inside the admin interface.
  • Manage in WebUI: Log in as Admin ➜ Settings ➜ Backup & Restore ➜ enter password to Reveal or Regenerate.
  • Restore on updates: If you migrate an old setup, copy the existing secret key from your legacy configuration and insert it as the "secret_key" value inside your config.json.

🌗 UI Appearance (Theme)

The appearance theme (Dark/Light) is managed in the settings panel and saved in config.json ("theme": "dark" or "light"). Dark theme is activated by default and loaded server-side. There is no manual theme toggler in the navigation sidebar.

  • /LLMWikiNG/settings?tab=theme – Update theme via the settings tab.
  • POST /LLMWikiNG/theme/set – directly set with value=dark|light (Authentication required).
  • Restore default: change "theme" value in config.json back to "dark".

🐋 Docker & UGreen NAS

The Docker configuration is explicitly optimized for UGreen NAS (UGOS) to guarantee out-of-the-box compatibility. The file structure is automatically set up under volume1/docker/llmwiking/.

UGreen NAS (UGOS) Setup:

When importing the image into the UGreen Container Manager, the following volumes are pre-defined:

  • /app/data ➜ /volume1/docker/llmwiking/data (User & API-Key DBs)
  • /app/wikis ➜ /volume1/docker/llmwiking/wikis (Wiki directories)
  • /app/raw ➜ /volume1/docker/llmwiking/raw (Raw sources)
  • /app/output_docs ➜ /volume1/docker/llmwiking/output_docs (Document exports)

Local Deployment & Development:

To run the Docker setup locally or on another host without file-over-directory mount conflicts (specifically with config.json), use the DOCKER_VOLUME_BASE environment variable:

  1. Create a local .env file in the project root: 
    DOCKER_VOLUME_BASE=.
  2. Start the containers: 
    docker compose up --build

🎨 Frontend (Tailwind CSS v4)

The styling is built via a CSS-First compiler pipeline (using @theme directives with oklch values, Dark mode media-controls, and responsive parameters):

cd frontend && npm install && npm run build

base.html links to compiled static/css/tailwind-build.css. All active JavaScript logic is written as pure ES modules under static/js/.
